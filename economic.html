Economic Model

The Whytegod Protocol uses WGD as a minimal economic utility to introduce cost, prevent abuse, and support long-term protocol stability.

Purpose of WGD

  • Anti-spam mechanism
  • Anchoring cost unit
  • Neutral usage signal

Anchoring Cost

Creating a permanent record requires a fixed amount of WGD. The token is consumed as part of the anchoring process.

  • Introduces economic friction
  • Prevents mass abuse
  • Encourages intentional usage

Token Members

A WGD Token Member is any individual or entity that holds WGD for the purpose of interacting with the Whytegod Protocol.

What Token Members Can Do

  • Pay anchoring costs
  • Create protocol records
  • Use verification tools
  • Interact with public protocol utilities

What Token Members Cannot Do

  • No governance or voting rights
  • No protocol control
  • No ownership claims
  • No administrative privileges

Holding WGD provides access to protocol usage only. It does not grant authority, influence, or decision-making power.

What WGD Is Not

  • Not a governance token
  • Not an investment contract
  • Not a profit-sharing instrument
  • Not a representation of ownership

Supply Philosophy

The protocol does not manage price, promise appreciation, or enforce economic outcomes. All market behavior exists outside the protocol definition.

Whytegod defines usage rules only. Economic interpretation and value perception are external and voluntary.
