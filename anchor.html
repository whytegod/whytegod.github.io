Anchor a Record

Anchoring is the act that gives a Whytegod Record ID (WRID) real force. Until anchored, a record is only a claim.

What Gets Anchored

  • Only the WRID hash
  • No original data
  • No identities
  • No permissions

How Anchoring Works

  1. Generate a WRID using the Hash tool
  2. Publish the WRID to an immutable public system
  3. Save the external reference permanently

Valid Anchor Targets

  • Bitcoin OP_RETURN
  • Ethereum calldata
  • Public timestamping services
  • Public append-only logs
  • Signed public statements
Whytegod Record Status

WRID: 
State: Anchored
Anchor Type: External Public System
Anchor Reference: 
Timestamp: Verifiable

Once anchored, a record cannot be altered or revoked.

Why This Matters

This is the same transition Bitcoin makes: from unsigned data to irreversible history. Meaning comes from public finality — not from the hash itself.

Whytegod does not run nodes, store anchors, or verify transactions. The protocol defines structure only.
