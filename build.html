Who This Is For

This page is intended for developers, researchers, auditors, and infrastructure builders.

Whytegod does not seek marketers, influencers, or short-term speculators.

How to Participate

  • Run a testnet node and observe validator behavior
  • Audit monetary and issuance logic
  • Test wallet privacy assumptions
  • Review Records for consistency and completeness

Contribution Areas

  • Core protocol implementation
  • Wallet tooling and recovery flows
  • Privacy analysis and adversarial modeling
  • Documentation and specification clarity

Rules of Engagement

  • No hype-driven contributions
  • No governance capture attempts
  • No shortcuts around protocol rules
  • Correctness is valued over speed

Development Philosophy

Whytegod evolves cautiously. Features are accepted only when they preserve monetary integrity, privacy guarantees, and long-term simplicity.
