Using the Protocol

The Whytegod Protocol is used to create permanent, verifiable records without accounts, identity, governance, or trust assumptions.

1. Create a Record Hash

A record begins with data. This data is never uploaded to the protocol. Instead, a cryptographic hash of the data is generated locally.

  • Choose any data (file, document, text, dataset)
  • Generate a SHA-256 hash of the data
  • The hash becomes the Record ID

The original data remains private. Only the hash is anchored.

2. Anchor the Record

The record hash is anchored by publishing it through the Whytegod Protocol anchoring mechanism.

  • The hash is included as an immutable reference
  • An on-chain or neutral anchor confirms permanence
  • No ownership or identity is recorded

Once anchored, the record cannot be altered, revoked, or censored.

3. Verify a Record

Any party can independently verify a record at any time.

  • Recompute the hash from the original data
  • Compare it with the anchored Record ID
  • Matching hashes confirm authenticity

Role of WGD

WGD exists to support anchoring demand and long-term sustainability. It does not grant control, governance rights, or special privileges.

  • Used for anchoring operations
  • Neutral utility only
  • No protocol authority attached

What the Protocol Does Not Do

  • No user accounts
  • No file storage
  • No identity system
  • No governance or voting
  • No record deletion
