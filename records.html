Record Layer

1. Record Definition

A record is a cryptographic commitment representing the existence of data at a specific point in time. The protocol never stores raw data — only proofs.

2. Record Format

Each record consists of:

  • Hash commitment (content fingerprint)
  • Timestamp anchor
  • Protocol version marker

Example structure: { hash || time || ruleset }

3. Anchoring Rules

Records are anchored immutably. Once confirmed, a record cannot be altered, replaced, or removed. Re-anchoring requires a new commitment.

4. Verification

Anyone can independently verify a record by recomputing the hash and comparing it against the anchored proof. No permission or identity is required.

5. Privacy Model

Records reveal nothing about the underlying data. Ownership, intent, and identity are intentionally absent from the system.

6. Lifecycle

Records persist indefinitely. Their value increases with network longevity, reinforcing permanence as a measurable property rather than a promise.
