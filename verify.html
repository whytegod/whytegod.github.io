Verify a Record

Verification confirms that a piece of data existed in a specific form at or before a given time. No permission, account, or token is required.

What You Need

  • The original data
  • The Record ID (hash)
  • The public anchor reference

Step-by-Step Verification

  1. Recreate the exact original data
  2. Generate its hash using the same algorithm
  3. Compare the result to the Record ID
  4. Locate the Record ID in the public anchor
  5. Confirm timestamp and immutability

What a Match Means

  • The data has not been altered
  • The data existed before the anchor time
  • No trust in Whytegod is required

What a Mismatch Means

  • The data was modified
  • The wrong hashing method was used
  • The wrong record was referenced

Who Can Verify

Anyone. Verification does not depend on Whytegod servers, accounts, APIs, or permission.

Tools

  • Offline hash generators
  • Open-source cryptographic libraries
  • Public blockchain explorers
  • Public timestamp logs

What Whytegod Does Not Do

  • Confirm identity
  • Judge meaning or intent
  • Resolve disputes
  • Store verification results

Verification proves integrity and time — not authorship, ownership, or legality.
