Whytegod Testnet

The Whytegod testnet exists to validate correctness, privacy, and protocol discipline before any mainnet deployment.

Home Specification Protocols Records

Purpose

The testnet is not a marketing environment. It is a controlled adversarial space designed to expose flaws in monetary logic, privacy guarantees, and validator behavior.

Core Rules

  • All monetary logic mirrors mainnet behavior
  • Wert has no real-world value
  • Privacy mechanisms are enforced by default
  • No privileged roles or override keys exist

Participation

Participation is permissionless. Validators, wallet developers, auditors, and researchers are encouraged to interact with the network under realistic conditions.

Testnet Phases

Phase 0 — Genesis Validation

Genesis parameters, fixed supply enforcement, Wert denomination, and issuance logic are verified.

  • Genesis record creation
  • Emission curve validation
  • Protocol invariants locked

Phase 1 — Validator Discipline

Validators operate under Proof of Integrity rules with simulated slashing and uptime requirements.

  • Fault injection testing
  • Slashing enforcement
  • Validator rotation analysis

Phase 2 — Wallet & Privacy Stress

Wallet behavior and privacy guarantees are tested against transaction graph analysis and metadata inspection.

  • One-time address enforcement
  • View key separation validation
  • Balance inference resistance

Phase 3 — Integration Simulation

External system interaction is simulated without granting state modification rights to third parties.

  • Settlement referencing
  • Observation anchoring
  • Protocol boundary testing

Incentives & Recognition

Testnet participation may be acknowledged through records, contributor recognition, or eligibility for future distribution. No speculative promises are made.

Exit Criteria

The testnet concludes only when all phases complete without unresolved critical failures. Mainnet deployment is contingent on protocol correctness, not timelines.
