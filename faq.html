Frequently Asked Questions

This page answers common questions about the Whytegod Protocol in clear, non-technical language.

What is Whytegod?

Whytegod is a neutral protocol for creating and verifying permanent records using cryptographic hashes and public anchors.

What problem does it solve?

It provides a way to prove that specific data existed at a specific time without revealing the data itself.

Does Whytegod store my data?

No. The protocol never stores user data. Only cryptographic hashes are used.

What is a record?

A record is a cryptographic hash derived from data. The original data always stays with the user.

What is anchoring?

Anchoring is the act of publishing a record hash to an immutable public system such as a blockchain or append-only log.

Can records be changed or deleted?

No. Once a record is anchored, it is immutable. This permanence is a core property of the protocol.

What is verification?

Verification is the process of recomputing a hash and comparing it with an anchored record. Verification is always free and permissionless.

What is the token used for?

The token is a utility mechanism used to prevent spam, rate-limit anchoring, and support protocol sustainability.

Is the token an investment?

No. The token does not represent equity, ownership, profit rights, or dividends.

Do I need the token to verify records?

No. Verification does not require a token. Tokens are only relevant for anchoring actions.

Who controls Whytegod?

No one. The protocol has no administrators, no override keys, and no centralized authority.

Can the protocol rules change?

Existing rules and records cannot change. New versions may be proposed, but adoption is voluntary.

Is Whytegod tied to any blockchain?

No. The protocol is blockchain-agnostic and can anchor records on any immutable system.

Is this legal?

Whytegod defines technical rules only. Legal interpretation depends on jurisdiction and use case.

What happens if the website disappears?

Nothing breaks. Verification depends only on hashes and public anchors, not on this website.

Whytegod provides tools for verification — responsibility for usage rests with the user.
