Whytegod Protocols

This document describes how the Whytegod network interfaces with external systems, internal modules, and protocol-level integrations without compromising core monetary or privacy guarantees.

Home Specification Records

Protocol Philosophy

Whytegod protocols are intentionally minimal, composable, and non-invasive. Integration does not imply dependency, and no external system gains control over core network behavior.

Integration Model

External systems may interact with Whytegod through defined interfaces for settlement, verification, and anchoring. These interactions are one-directional by default and cannot modify protocol state.

Upgrade Discipline

Whytegod does not perform silent upgrades. Any protocol-level change must be explicitly defined, categorized, and recorded. Core monetary and privacy rules are excluded from upgrade scope.
